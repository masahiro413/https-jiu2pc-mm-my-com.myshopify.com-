# CLAUDE.md — Shopify DVDストア 開発ガイド

## プロジェクト概要

西洋時代劇・鉄道車窓DVDを販売する個人EC。Shopify + Horizonテーマベース。

## 技術スタック

- **プラットフォーム**: Shopify (Online Store 2.0)
- **テーマ**: Horizon（無料、OS2.0対応）
- **カスタマイズ**: CSS変数上書き + Liquid セクション編集
- **決済**: Shopify Payments（クレジットカード）
- **言語**: 日本語

## 開発ルール

### テーマカスタマイズ方針
- Horizonの`settings_schema.json`カスタマイズを最優先（コード変更より設定変更を先に試す）
- CSS変更は`assets/custom.css`に集約。テーマファイルを直接書き換えない
- Liquid変更はセクションファイル単位で行い、スニペットは`snippets/`に切り出す
- `layout/theme.liquid`の変更は最小限に留める

### デザイン原則（Netflixダークトーン）
- カラースキーム: ダーク背景（`#141414`）、アクセント赤（`#E50914`）、テキスト白
- 商品画像は16:9またはDVDジャケット比率（約0.7:1）で統一
- フォント: 日本語 Noto Sans JP、見出しは太め（700）

### 在庫管理の注意点
- 商品は最大10枚/タイトル。在庫切れ時の自動購入ブロックを必ず有効化
- Shopify管理画面 > 商品 > 在庫追跡を「Shopifyで追跡」に設定すること
- 残り3枚以下は「残りわずか」表示（Liquidで`variant.inventory_quantity`を参照）

### 運用自動化（最優先）
- 注文確認メール・発送通知メールはShopify標準テンプレートを活用
- 追跡番号はShopify管理画面「注文 > フルフィルメント」から登録
- 手動作業が発生するのは「梱包・発送・追跡番号入力」の3工程のみにする

## ファイル構成メモ

```
sections/
  header.liquid         # ナビゲーション
  featured-product.liquid  # 商品詳細ヒーローセクション
  collection-*.liquid   # 商品一覧
  blog-*.liquid         # ブログ
snippets/
  product-card.liquid   # カードUI（一覧・レコメンド共用）
  low-stock-badge.liquid # 残りわずかバッジ
assets/
  custom.css            # テーマ上書きCSS（ここに集約）
```

## 完成条件

**1件の実際の購入が完了した時点で完成とみなす。**
機能網羅より「買える状態」を最優先に実装する。

## 2. ターゲットユーザーと購買動機

- **流入経路**: SNS（X/Instagram等）→ スマートフォン経由
- **購買動機**: 「この商品でなければならない」（代替不可のニッチコンテンツ）
- **デバイス**: 顧客はスマートフォン主体。レスポンシブ対応必須
- **購買判断**: SNSで認知→その場で購入決断（衝動購買型）

**設計上の含意**: ページ遷移数を最小化し、商品詳細→カート→チェックアウトの導線をモバイルで完結させる。

## 3. デザイン仕様

### ビジュアル方向性
- **テーマ**: シネマティック・ヒーローティック
- **参照**: Netflixのタイトル詳細ページ（ダークトーン・ドラマチックなヒーロー画像・カードUI）

### カラースキーム
| 役割 | カラーコード |

## 4. ページ仕様

### 4.1 トップページ（Home）

| セクション | 内容 | 優先度 |
|-----------|------|--------|
| ヒーローバナー | 全幅の劇的な映像/画像＋キャッチコピー＋CTAボタン | 必須 |
| 注目商品 | 現在販売中タイトルを2〜4点カード表示 | 必須 |
| ブランドストーリー | 1〜2文のコンセプト文＋Aboutへのリンク | 必須 |
| 新着ブログ記事 | 最新2〜3記事のサムネ＋タイトル | 任意（後回し可） |

### 4.2 商品一覧ページ（Collection）

- カテゴリー（西洋時代劇 / 鉄道車窓）でフィルタリング
- 在庫切れ商品は「売り切れ」バッジ表示＋グレーアウト（購入不可）
- ソート: 新着順・価格順
- カードUI: ジャケット画像・タイトル・価格・在庫状況

### 4.3 商品詳細ページ（Product）

**必須要素:**
- ヒーロー画像（ジャケット＋場面写複数枚スライダー）
- 商品名・価格
- 商品説明文（収録内容・収録時間・画質・字幕有無）
- 在庫残数（残り3枚以下で「残りわずか X 枚」表示）
- 「カートに入れる」ボタン（在庫切れ時は無効化）
- 配送情報（発送目安・送料）

**任意要素（後回し可）:**
- 関連商品（同カテゴリの他タイトル）
- レビュー欄（Product Reviews App）

### 4.4 カートページ（Cart）

- 商品一覧（画像・タイトル・価格・数量変更・削除）
- 小計金額
- 「ご購入手続きへ」ボタン（Shopify標準チェックアウトへ遷移）
- 注記: ドロワーカート（スライドイン型）は後回し可

### 4.5 Aboutページ

- 運営者の想い・コンセプト文（300〜500文字程度）
- 商品へのこだわり（製作背景・選定基準）
- 運営者情報（名前または屋号程度でよい）

### 4.6 ブログページ（Blog）

- 記事一覧: サムネイル・タイトル・投稿日・カテゴリタグ
- 記事詳細: 本文・関連記事リンク・SNSシェアボタン
- カテゴリ: 西洋時代劇 / 鉄道 / 商品レビュー 等
- 投稿頻度: 月数本以上（SEO・ファン育成目的）

### 4.7 お問い合わせページ（Contact）

- Shopify標準フォーム（名前・メール・件名・本文）
- 対応予定時間の明記（例: 「3営業日以内に返信」）
- FAQへのリンク（後回し可）

### 4.8 ポリシーページ

Shopify管理画面から生成・管理:
- 特定商取引法に基づく表記（法的必須）
- プライバシーポリシー
- 返品・返金ポリシー（物理メディアのため原則交換のみ）
- 配送ポリシー

---

## 5. 在庫・注文フロー仕様

### 在庫管理
- 在庫追跡: Shopifyによる自動追跡（`track_inventory: true`）
- 在庫切れ時: 購入ボタン自動無効化
- 残りわずか閾値: 3枚以下で警告表示

### 注文〜発送フロー
```
顧客が購入
  ↓
Shopify自動メール「注文確認」（即時）
  ↓
運営者が梱包・発送
  ↓
管理画面から追跡番号を入力・フルフィルメント完了
  ↓
Shopify自動メール「発送完了＋追跡番号」（自動送信）
```

**手動作業は「梱包・発送・追跡番号入力」の3工程のみ。**
